# 🚀 Sprint Review — LabControll 4.0

## 🎯 Objetivo da Sprint
Validar a proposta do LabControll 4.0, estruturando sua base técnica (backend, banco de dados e arquitetura), além de iniciar o desenvolvimento das interfaces e consolidar a documentação do projeto.

---

## 📦 Entregas da Sprint

Durante esta sprint, o time concentrou esforços na **fundação do sistema**, com foco em backend, arquitetura e design.

### 🧠 Backend (Principal Entrega)
- Desenvolvimento da lógica principal da **planta baixa interativa**
- Estruturação de regras para:
  - Monitoramento de ambientes
  - Status de salas e máquinas
  - Organização dos dados para visualização
- Criação da base da API para integração futura

---

### 🎨 Design (Figma)
- Desenvolvimento completo do design do sistema
- Criação de protótipos de:
  - Dashboard
  - Planta baixa interativa
  - Fluxos de autenticação
- Definição da identidade visual e experiência do usuário

---

### 🌐 Frontend Web
- Início da implementação da interface
- Desenvolvimento da **interface da planta baixa**
- Estrutura base para integração com backend

---

### 📱 Mobile
- Desenvolvimento das telas de autenticação:
  - Login
  - Cadastro
- Preparação para integração com API

---

### 🗄️ Banco de Dados
- Estruturação do banco de dados
- Modelagem inicial das entidades:
  - Usuários
  - Ambientes
  - Máquinas
  - Consumo energético

---

### 📄 Documentação
Foram produzidos artefatos essenciais para o projeto:

- Documento da demanda (SAGA)
- Mapa de integração vertical e horizontal
- Arquitetura geral do sistema
- Backlog inicial do produto

---

## 🧠 Decisões Arquiteturais

O time definiu uma arquitetura baseada em camadas, alinhada à Indústria 4.0:

- Separação entre **IoT, Backend, Frontend e Cloud**
- Uso de **API REST** para comunicação
- Banco relacional (**PostgreSQL**) para consistência dos dados

### 🔍 Justificativas:
- Facilidade de escalabilidade
- Organização do sistema
- Suporte a múltiplas interfaces (web e mobile)
- Preparação para integração com sensores reais

---

## 💡 Valor Gerado na Sprint

Mesmo sendo uma sprint inicial, foram gerados valores importantes:

### 🏗️ Base Sólida do Sistema
- Backend estruturado para evolução rápida
- Banco de dados modelado corretamente

### 🎯 Clareza do Produto
- Design completo no Figma
- Definição clara das funcionalidades

### ⚙️ Redução de Riscos
- Arquitetura bem definida
- Documentação consolidada

### 🚀 Aceleração das Próximas Entregas
- Interfaces já iniciadas
- Backend pronto para integração

# 🔁 Sprint Retrospective

Melhorar continuamente o processo do time.

---

## ✅ Pontos Positivos
- Boa divisão entre backend, frontend e design
- Entrega consistente de documentação
- Forte avanço na base técnica do sistema
- Organização do backlog inicial

---

## ❌ Pontos Negativos
- Integração entre áreas ainda limitada
- Dependência entre tarefas não mapeada claramente
- Falta de validação prática (execução real do sistema)

## 🔧 Pontos de Melhoria

- Planejar integrações desde o início
- Aumentar testes durante o desenvolvimento

---

## 📈 Ações para Próxima Sprint
